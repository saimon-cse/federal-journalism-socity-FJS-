2.  **Dependencies:** Ensure that tables with foreign keys are created *after* the tables they reference. Laravel's timestamp-based ordering usually handles this if you create migrations in a logical sequence (e.g., `divisions` before `districts`, `users` before `user_profiles`).
3.  **Existing Migrations:** If you have existing `users` table migration (e.g., from Breeze/Jetstream), you'll need to *modify* that one or delete it and use the one provided below.
